VAR :
* Variables declared with var have function scope or global scope, but not block scope.
* If a var variable is declared inside a function, it is accessible only within that function.
* If a var variable is declared outside any function, it becomes a global variable and is accessible throughout the entire script.
* var variables are subject to hoisting, meaning their declarations are moved to the top of their scope during the compilation phase, but not their assignments.

LET :
* Variables declared with let have block scope, meaning they are accessible only within the block (enclosed by curly braces) in which they are declared.
* Unlike var, let variables are not hoisted to the top of their scope. This means you cannot access them before the line where they are declared.
* let allows you to create variables that are limited to the scope of a block statement, expression, or on a global level.

CONST :
* Variables declared with const also have block scope like let.
* The value of a const variable cannot be changed once it's assigned. It is a constant.
* Similar to let, const variables are not hoisted to the top of their scope.
* However, it's important to note that while the variable itself cannot be reassigned, if the value is an object or array, its properties or elements can be mutated.

NULL:
API Responses: In APIs, NULL may be used to represent the absence of a value in a response. For example, a JSON field might be explicitly set to null to indicate that there is no value for that field.
Database Integration: APIs often interact with databases, where NULL values are commonly used to represent missing or unknown data. API responses may mirror this behavior by returning NULL for fields that have no value.

UNDEFINED:
JavaScript Variables and Properties: In JavaScript, UNDEFINED is a special value that indicates a variable has been declared but not assigned a value, or that a property does not exist in an object.
Function Returns: In JavaScript, functions that do not explicitly return a value return UNDEFINED. This is often seen when a function terminates without encountering a return statement.

RESET API :
REST APIs communicate through HTTP requests to perform standard database functions like creating, reading, updating and deleting records (also known as CRUD) within a resource.
For example, a REST API would use a GET request to retrieve a record. A POST request creates a new record.
